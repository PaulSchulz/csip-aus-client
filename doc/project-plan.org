** Outline
To build a CSIP-AUS client on a microcontroller like the ESP32‑S3, the key
challenge is that CSIP‑AUS (Consumer Switching Interface Protocol – Australia)
is normally implemented on full Linux systems (gateways, EV chargers, batteries,
etc.) and relies on:

- HTTPS/TLS
- JSON APIs
- OAuth2 / token authentication
- Continuous internet connectivity

An ESP32-class MCU can do it, but the design must be very lightweight and
usually only implements the client subset of the spec.

Below is a practical architecture used in embedded implementations.

*** CSIP-AUS Overview (What client must do)
A CSIP-AUS Client Energy Resource (CER) communicates with an Aggregator.

Typical functions:

| Function                | Direction       |
|-------------------------+-----------------|
| Register device         | Client → Server |
| Heartbeat/status        | Client → Server |
| Receive dispatch limits | Server → Client |
| Acknowledge commands    | Client → Server |

Typical messages:
#+begin_src 
POST /cer/register
POST /cer/heartbeat
POST /cer/event
GET  /dispatch/instructions
#+end_src

Payloads are JSON.

Example dispatch message:
#+begin_src json
{
  "eventId": "123456",
  "limitW": 3500,
  "startTime": "2026-03-12T10:00:00Z",
  "duration": 900
}
#+end_src

** Basic CSIP Client

#+end_src

src/csip_client.cpp
#+begin_src cpp :tangle ../src/csip_client.cpp :main no :comment link
#include "csip_client.h"
#include "config.h"

#include <HTTPClient.h>
#include <ArduinoJson.h>

bool CSIPClient::begin()
{
    client.setInsecure();
    return true;
}

bool CSIPClient::registerCER()
{
    HTTPClient https;

    String url = String(CSIP_SERVER) + "/cer/register";

    https.begin(client, url);
    https.addHeader("Content-Type", "application/json");

    StaticJsonDocument<256> doc;

    doc["deviceId"] = DEVICE_ID;
    doc["type"] = "load";

    String body;
    serializeJson(doc, body);

    int code = https.POST(body);

    Serial.printf("Register response: %d\n", code);

    https.end();

    return code == 200;
}

bool CSIPClient::heartbeat()
{
    HTTPClient https;

    String url = String(CSIP_SERVER) + "/cer/heartbeat";

    https.begin(client, url);
    https.addHeader("Content-Type", "application/json");

    StaticJsonDocument<128> doc;

    doc["deviceId"] = DEVICE_ID;

    String body;
    serializeJson(doc, body);

    int code = https.POST(body);

    Serial.printf("Heartbeat: %d\n", code);

    https.end();

    return code == 200;
}

bool CSIPClient::getDispatch()
{
    HTTPClient https;

    String url = String(CSIP_SERVER) + "/dispatch";

    https.begin(client, url);

    int code = https.GET();

    if (code != 200)
    {
        https.end();
        return false;
    }

    String payload = https.getString();

    StaticJsonDocument<512> doc;
    deserializeJson(doc, payload);

    int limit = doc["limitW"];

    Serial.printf("Dispatch limit: %dW\n", limit);

    https.end();

    return true;
}

#+end_src

*** Dispatch Polling Example
#+begin_src cpp :tangle ../src/dispatch_handler.cpp :main no :comment link
bool CSIPClient::getDispatch()
{
    HTTPClient https;

    https.begin(client, "https://api.csip.example.com/dispatch");

    int code = https.GET();

    if(code != 200)
        return false;

    String payload = https.getString();

    StaticJsonDocument<512> doc;
    deserializeJson(doc, payload);

    int limit = doc["limitW"];

    applyPowerLimit(limit);

    https.end();

    return true;
}
#+end_src

*** Main Loop

src/main.cpp
#+begin_src sh :tangle ../src/main.cpp :main no :comment link
#include <Arduino.h>
#include <WiFi.h>

#include "config.h"
#include "csip_client.h"

CSIPClient csip;

unsigned long lastHeartbeat = 0;

void connectWiFi()
{
    WiFi.begin(WIFI_SSID, WIFI_PASSWORD);

    Serial.print("Connecting");

    while (WiFi.status() != WL_CONNECTED)
    {
        delay(500);
        Serial.print(".");
    }

    Serial.println();
    Serial.println("WiFi connected");
}

void setup()
{
    Serial.begin(115200);

    connectWiFi();

    csip.begin();
    csip.registerCER();
}

void loop()
{
    if (millis() - lastHeartbeat > HEARTBEAT_INTERVAL)
    {
        csip.heartbeat();
        csip.getDispatch();

        lastHeartbeat = millis();
    }
}
#+end_src

*** Important ESP32 Considerations
**** TLS RAM usage

HTTPS consumes ~50-70 KB RAM.

For ESP32-S3 this is fine, but you should:
#+begin_src 
CONFIG_MBEDTLS_SSL_MAX_CONTENT_LEN=4096
#+end_src

**** JSON size
Keep documents small.
#+begin_src 
StaticJsonDocument<512>
#+end_src
Avoid dynamic allocation.

**** Certificate validation
For production you should load the CSIP server root certificate.
#+begin_src 
client.setCACert(root_cert);
#+end_src

*** Strong Recommendation (Industry Practice)
Most commercial systems do not implement CSIP directly on the MCU.

Instead they use:
#+begin_src 
MCU → MQTT → Edge Gateway → CSIP
#+end_src

Example gateway platforms:
- Home Assistant
- Node‑RED
- custom Linux gateway

Reasons:
- easier certificate management
- easier OTA updates
- easier debugging
